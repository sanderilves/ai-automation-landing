Case study

Real work. Already running.

Built as a production-ready prototype for a Baltic fintech — demonstrating what a compliant, automated debt collections workflow looks like in practice.

The problem
  • Specialists manually checked the CRM every day to decide who to call
  • No automated tracking of whether a customer had already been contacted that day
  • Every call outcome had to be logged by hand — slow and inconsistent
  • No audit trail if a compliance question came up later
What we built
  • Automatically identifies which customers need to be contacted each day
  • Checks the right contact window before any call is made
  • Sends the call to an AI voice agent — no specialist needed for routine cases
  • Escalates to a human automatically when the case needs judgment
  • Logs every interaction for compliance without any manual effort
